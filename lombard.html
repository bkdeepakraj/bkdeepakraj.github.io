Lombard E-sign (Adobe E-Sign)

UX Design

This Project is under NDA. Actual Screens are not part of the Case Study.

Project Summary

Lombard E-sign project is part of Asset Management in Lombard. Assets like Vehicles, Agricultural, Marine, Technology, and others assets are financed to the customers or companies. During the Quote & finance for the asset, Lombard agent generally follow manual process to get the documents verified and signed (wet-sign) by customers. Lombard E-sign is a process where all the documents are verified and signed by customers digitally.

The UX Challenge

The challenge was to understand the Asset Management domain and get used to the technical terms used in asset finance documentation . Based on the domain, understand the specific users or customers who need finance for range of assets (Vehicles, Agricultural, Marine, Technology, and others assets) and the needs/goals of each user.

The UX Approach

The project was managed through Agile process. The UX process(Iteration) as follows, Collect Requirements from Stakeholders or Project Team > Understand and send queries to the project team > Identify use cases and problem statement > Create Task Flows, User Journey and Personas in Adobe XD > Create UX Prototype based on the use cases > Review with project team > Identify and fix issues based on feedback if any from the project team > Deliver the prototype to the team

Stakeholders

Business Analyst, Developers, Front-end Developers and Scrum Master

UX Principles

Context: What is the context of each flow or the goal the user wants to achieve. The layout, template, menu and UI elements placed in each screen of the UX workflow are relevant and does it help user to achieve the goal.

Goal Setting: Does the Intended workflow helps user to achieve the goal efficiently and effectively.

Customer Statisfication: Does the design matches user needs and improves customer satisfication.

Followed some of the below Heuristic(Norman-Jakob Neilsen) principles,

Consistency: All the fonts, words, phrases color, size of the UI elements, UI patterns used is consistent and mean same thing in each screen of the workflow.

Visibility of the system status: Designs should keep users informed about what is going on, through appropriate, timely feedback.

Match between System and the Real World: The design should speak the users' language. Use words, phrases, and concepts familiar to the user, rather than internal jargon.

User Control and Freedom: Users often perform actions by mistake. They need a clearly marked "emergency exit" to leave the unwanted action.

Error Prevention: Good error messages are important, but the best designs carefully prevent problems from occurring in the first place.

Recognition Rather Than Recall: Minimize the user's memory load by making elements, actions, and options visible. Avoid making users remember information.

Flexibility and Efficiency of Use: Shortcuts hidden from novice users may speed up the interaction for the expert user.

Aesthetic and Minimalist Design: Interfaces should not contain information which is irrelevant. Every extra unit of information in an interface competes with the relevant units of information.

Recognize, Diagnose, and Recover from Errors: Error messages should be expressed in plain language (no error codes), precisely indicate the problem, and constructively suggest a solution.

Help and Documentation: It’s best if the design doesn’t need any additional explanation. However, it may be necessary to provide documentation to help users complete their tasks.
