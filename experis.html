Job Search Portal, offering white(service industry) collar jobs

Project Name

Experis.com Website Re-design

Project Tagline

A site for searching white collar Jobs where Users can create profile, search for jobs and set preferences for the job and apply for jobs.

Project Summary

Experis.com website is Job listing site where hundreds of blue collar jobs are listed in the site. The users can search for relevant jobs and apply for jobs. My role is to create UI and visual design for the website in collaboration with Art director of Manpower group.

Client Name

Manpower Group

Timeframe

Jan 2016 to March 2016, Wipro (Onsite & Offsite)

My Major Tasks & Responsibilities

My task was to create UX workflow and UI screens based on the business requirement of Manpower Group. Create flow for job search, list jobs, retrieve the job information and apply for the Job. Create a sign up workflow for users to register with the site.

Platforms

WebApp, Bootstrap Responsive Site.

Key Performance Metrics

Users Engagement with website, Retrieval of correct Job information and number of jobs people apply through the site.

Team Members & Collaborators

Business Analyst(SME's), Visual Designer, UI Developer, Client, Project Manager

The UX Challenge

The project was to redesign to responsive site by using Fluid Grid System, Fluid Image Use and Media Queries. UX principles such uniform design, SEO friendly and minimalism in design were followed.

The UX Approach

Through design iteration the existing screens were re-designed to responsive design. To make it responsive, I collaborated with Bootstrap developers and the site was revamped into responsive site.

Visual Design

The client had created style guide/Design System for the site. The UI was designed to match the style guide before making it responsive.

RWD Principles

Consistency: All the UI elements designed for both mobile and desktop should have consistency while using colors, font-size, margin and padding across ui components, width, and height of buttons and form fields.

Page elements reshuffle: As Viewport shrinks the elements reshuffle based on the breakpoints. The features visibility which are most important while switching from each breakpoint were maintained seamlessly.

Space Constraint: The viewport width is small for mobile when compared to desktop. The text, buttons and form fields accessability has to be maintained by specifying correct font-size for readability, width and height of buttons & form fields has to be 40px to 45px when the breakpoint for mobile is considered.

Navigation: The Navigation has to be clear considering the mobile and tablet experience. Providing cues to the user where he is currently in the flow(displaying header text), what the user can do(displaying actions he can take on the current screen), where can he go from the specific screen(displaying appropriate nav menu) and how he got to a specific screen(displaying breadcrumb as trail). accordance's and cues while navigating the website was consistently maintained throughout the website.

Responsive Layout

The website was designed responsive and coded with Bootstrap
