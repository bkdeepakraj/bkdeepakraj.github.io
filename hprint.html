HP Print Design: Workflows for Printer Control Panel

User Experience Design Workflows for Control Panel

This Project is under NDA. Actual Screens are not part of the Case Study.

Project Summary

HP's division Print Design create and handle workflows for HP Printers(control panel - touch screen). The team consist of UX Designers, Pattern team, IA Team and Firmware team. I joined the Print Design Team in HP as UX Designer. I was assigned to create UX workflow for the feature Tools. Under Tools menu Reports, Maintenance, Troubleshooting and Service workflows has to be created.

The UX Challenge

The challenge was to create workflows by adhering to requirements constrained to HP Printer(Inkjet and Laser Printers) ranging from small, medium and enterprise printers and adhering to HP Printer Patterns created by Pattern team and collaborating with Firmware team

The UX Approach

The project was managed through Agile process. The UX process as follows, Collect Requirements > Understand and send queries to the project team > identify use cases > Create Task Flows in Axure > Create UX Workflows based on the Use cases and use layout, template, UI elements from HP Print Patterns > Review with project Team > Identify issues based on feedback if any from the project team > Release the workflow to the team > Tech lead and Firmware team implement the workflow inside the control panel of the printer.

Stakeholders

Tech Leads and Firmware Team

RWD Principles

Consistency: All the UI elements designed for both mobile and desktop should have consistency while using colors, font-size, margin and padding across ui components, width, and height of buttons and form fileds.

Page elements reshuffle: As Viewport shrinks the elements reshuffle based on the breakpoints. The features visiblity which are most important while switching from each breakpoint were maintained seamlessly.

Space Constraint: The viewport width is small for mobile when compared to desktop. The text, buttons and form fields accessbility has to be maintained by specifying correct font-size for readability, width and height of buttons & form fields has to be 40px to 45px when the breakpoint for mobile is considered.

Navigation: The Navigation has to be clear considering the mobile and tablet expierence. Providing cues to the user where he is currently in the flow(displaying header text), what the user can do(displaying actions he can take on the current screen), where can he go from the specific screen(displaying appropriate nav menu) and how he got to a specific screen(displaying breadcrumb as trail). Affordances and cues while navigating the website was conistently maintained throughout the website.

Responsive Layout

The website was designed responsive and coded with Bootstrap
