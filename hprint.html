HP Print Design: Workflows for Printer Control Panel

UX Design

This Project is under NDA. Actual Screens are not part of the Case Study.

Project Summary

HP's division Print Design create and handle workflows for HP Printers(control panel - touch screen). The team consist of UX Designers, Pattern team, IA Team and Firmware team. I joined the Print Design Team in HP as UX Designer. I was assigned to create UX workflow for the feature Tools. Under Tools menu Reports, Maintenance, Troubleshooting and Service workflows has to be created.

The UX Challenge

The challenge was to create workflows by adhering to requirements constrained to HP Printer(Inkjet and Laser Printers) ranging from small, medium and enterprise printers and adhering to HP Printer Patterns created by Pattern team and collaborating with Firmware team

The UX Approach

The project was managed through Agile process. The UX process(Iteration) as follows,

  1. Collect Requirements from Stakeholders or Project Team
    2. >
  2. Understand and send queries to the project team
  3. Identify use cases
  4. Create Task Flows in Axure
  5. Create UX Workflows based on the Use cases and use layout, template, UI elements from HP Print Patterns
  6. Review with project Team
  7. Identify and fix issues based on feedback if any from the project team
  8. Release the workflow to the team
  9. Tech lead and Firmware team implement the workflow inside the control panel of the printer.

Stakeholders

Tech Leads and Firmware Team

UX Principles

Context: What is the context of each flow or the goal the user wants to achieve. The layout, template, menu and UI elements placed in each screen of the workflow are relevant and does it help user to achieve the goal.

Goal Setting: Is the Intended workflow(i.e Tools, Print, Scan, Copy, Sttings ) helps user to achieve the goal efficiently and effectively.

Customer Statisfication: Does the design matches user needs and improves customer satisfication.

Followed some of the below Heuristic(Norman-Jakob Neilsen) principles,

Consistency: All the fonts, words, phrases color, size of the UI elements, HP design patterns used is consistent and mean same thing in each screen of the workflow.

Visibility of the system status: Designs should keep users informed about what is going on, throughappropriate, timely feedback.

Match between System and the Real World: The design should speak the users' language. Use words, phrases, and concepts familiar to the user, rather than internal jargon.

User Control and Freedom: Users often perform actions by mistake. They need a clearly marked "emergency exit" to leave the unwanted action.

Error Prevention: Good error messages are important, but the best designs carefully prevent problems from occurring in the first place.

Recognition Rather Than Recall: Minimize the user's memory load by making elements, actions, and options visible. Avoid making users remember information.

Flexibility and Efficiency of Use: Shortcuts hidden from novice users may speed up the interaction for the expert user.

Aesthetic and Minimalist Design: Interfaces should not contain information which is irrelevant. Every extra unit of information in an interface competes with the relevant units of information.

Recognize, Diagnose, and Recover from Errors: Error messages should be expressed in plain language (no error codes), precisely indicate the problem, and constructively suggest a solution.

Help and Documentation: It’s best if the design doesn’t need any additional explanation. However, it may be necessary to provide documentation to help users complete their tasks.
